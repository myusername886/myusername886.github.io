C 语言的标准

K&R C

K&R C,也叫经典 C, 旧的 C, 传统的 C, 以《The C Programming Language》【Brain Kernighan, Dennis Ritchie】(第 1 版) 作为 C 的标准, 取两个作者的姓的第一个字母称为 K&R 标准.

书的附录中定义了 C 语言, 未定义 C 库, 以 UNIX 实现提供的库作为标准库.

C89

C89 即 ANSI C, 由美国国家标准局 (American National Standards Institute, 简称 ANSI) 制定并批准.

后来 ANSI 把这个标准提交到国际化标准组织 (International Organization for Standardization, 简称 ISO) , 1990 年被 ISO 采纳为国际标准, 称为 ISO C. 又因为这个版本是 1990 年发布的, 因此也被称为 C90.

ANSI C (C89) 与 ISO C (C90) 内容基本相同, 主要是格式组织不一样.

ANSI C, ISO C, C89, C90, ANSI/ISO C, C89/90 这些称谓都是一样的, 也叫 标准 C.

这个标准定义了 C 语言和 C 标准库.

C99

1995年 C 程序设计语言工作组对 C 语言进行了一些修改, 增加了新的关键字, 编写了新的库, 取消了原有的限制, 并于 1999年形成新的标准—— ISO/IEC 9899:1999 标准, 通常被称为 C99.

C11

C11 标准由国际标准化组织 (ISO) 和国际电工委员会 (IEC) 旗下的 C 语言标准委员会于 2011 年底正式发布.

C11 标准主要增加了以下内容:

  • 增加了安全函数, 例如 gets_s(), fopen_s() 等;
  • 增加了头文件以支持多线程;
  • 增加了头文件以支持 Unicode 字符集.

三个标准的主要差别:

ANSI C 新增了函数原型.

ANSI C 规定, 所有的局部变量 (函数内部的变量) 都必须定义在函数的开头位置, 在定义完所有变量之前不能有其它表达式.

ANSI C 规定局部变量要在函数开头定义, C99 取消了这个限制, 变量可以在任意位置定义.

ANSI C 的注释只有 /* */ 的块注释形式, C99添加了 // 行注释形式.

C99 允许在 for 循环条件里定义新变量, 这样的变量是块级变量, 作用域仅限于 for 循环内部.

主流编译器对标准的支持情况

目前常用的编译器都支持 ANSI C.

GCC 支持 C99, 微软编译器不完全支持.

支持 C11 标准的编译器有 GCC, LLVM/Clang, Intel C++ Compile 等. 微软编译器不支持 C11.
